# App Store Screenshot Specifications

Complete specifications for iOS App Store and Google Play Store screenshots.

## iOS App Store

### Required Sizes

You must provide screenshots for at least one of these device sizes. App Store Connect will scale for other devices.

| Device | Display Size | Screenshot Size (Portrait) | Screenshot Size (Landscape) |
|--------|-------------|---------------------------|----------------------------|
| iPhone 6.7" (Pro Max) | 6.7" | 1290 x 2796 px | 2796 x 1290 px |
| iPhone 6.5" (Plus/Max) | 6.5" | 1284 x 2778 px | 2778 x 1284 px |
| iPhone 6.1" (Pro) | 6.1" | 1179 x 2556 px | 2556 x 1179 px |
| iPhone 5.5" (Plus) | 5.5" | 1242 x 2208 px | 2208 x 1242 px |
| iPad Pro 12.9" | 12.9" | 2048 x 2732 px | 2732 x 2048 px |
| iPad Pro 11" | 11" | 1668 x 2388 px | 2388 x 1668 px |

### Screenshot Limits

- **Minimum**: 2 screenshots per device
- **Maximum**: 10 screenshots per device
- **File format**: PNG or JPEG
- **File size**: Max 500 MB per screenshot
- **Color space**: sRGB or Display P3

### iOS Best Practices

1. **First 3 screenshots are critical** - shown in search results
2. **Consistent design** across all screenshots
3. **Device frames** are optional but professional
4. **Text overlays** should be large and readable
5. **Show real app UI** - not mockups
6. **Portrait orientation** preferred for phones
7. **Landscape** acceptable for games/video apps

## Google Play Store

### Required Sizes

| Device Type | Screenshot Size | Aspect Ratio |
|-------------|----------------|--------------|
| Phone | Min: 320 x 320 px | 16:9 (recommended) |
| Phone | Max: 3840 x 3840 px | - |
| Phone | Recommended: 1080 x 1920 px | 16:9 |
| Tablet 7" | 1200 x 1920 px | - |
| Tablet 10" | 1800 x 2560 px | - |
| Chromebook | 1920 x 1080 px | 16:9 |
| TV | 1920 x 1080 px | 16:9 |
| Wear OS | 384 x 384 px | 1:1 |

### Screenshot Limits

- **Minimum**: 2 screenshots (phone)
- **Maximum**: 8 screenshots per device type
- **File format**: PNG or JPEG (24-bit, no alpha)
- **File size**: Max 8 MB per screenshot

## Screenshot Content Strategy

### Recommended Order

| Position | Content | Purpose |
|----------|---------|---------|
| 1 | Hero/Value Prop | Capture attention, main benefit |
| 2 | Key Feature 1 | Most popular/important feature |
| 3 | Key Feature 2 | Secondary differentiator |
| 4 | Social Proof | Build trust |
| 5 | Key Feature 3 | Additional value |
| 6 | Use Case | Show real-world application |
| 7 | Feature 4 | Depth of features |
| 8 | CTA/Download | Final push |

### Caption Best Practices

**Do**:
- Focus on benefits, not features
- Use action verbs
- Keep it concise (3-7 words)
- Make it readable at small size
- Test contrast and legibility

**Don't**:
- Use small fonts
- Overcrowd with text
- Use generic phrases
- Forget localization needs

### Caption Examples

| Feature | Bad Caption | Good Caption |
|---------|------------|--------------|
| Progress tracking | "Track your progress" | "See how far you've come" |
| Reminders | "Set reminders" | "Never miss a session" |
| Statistics | "View statistics" | "Celebrate your streaks" |
| Sleep sounds | "Sleep sounds" | "Fall asleep in minutes" |

## Technical Specifications Summary

### File Requirements

| Specification | iOS | Android |
|--------------|-----|---------|
| Format | PNG, JPEG | PNG, JPEG (24-bit) |
| Color depth | 24-bit | 24-bit, no alpha |
| Color space | sRGB, Display P3 | sRGB |
| Max file size | 500 MB | 8 MB |
| Transparency | Allowed (PNG) | Not allowed |

### Aspect Ratio Guidelines

| Ratio | Use Case | Example Size |
|-------|----------|--------------|
| 16:9 | Standard phone (portrait) | 1080 x 1920 |
| 9:16 | Landscape mode | 1920 x 1080 |
| 4:3 | iPad/Tablet | 2048 x 2732 |
| 1:1 | Wear OS | 384 x 384 |

## Quality Checklist

### Before Upload

- [ ] Correct dimensions for device type
- [ ] File size within limits
- [ ] No transparency (Android)
- [ ] Text readable at 50% zoom
- [ ] Consistent design language
- [ ] Brand colors used
- [ ] High-resolution assets
- [ ] No placeholder content
- [ ] No competitor references
- [ ] Compliant with store guidelines

### Localization

- [ ] All text translated
- [ ] Cultural appropriateness checked
- [ ] Local number/date formats
- [ ] Reading direction (RTL languages)
- [ ] Local visual preferences considered
