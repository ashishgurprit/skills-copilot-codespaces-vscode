# eawesome.com.au Brand Guidelines

## Brand Identity

**Tagline**: Expert App Development for Australian Startups
**Position**: Developer-focused app development expertise
**Author**: Ash Ganda

## Target Audience

### Primary
- **App founders** building their first or next product
- **Product managers** at Australian startups
- **Technical co-founders** making architecture decisions

### Secondary
- Mobile developers seeking best practices
- CTOs at app-focused companies
- Investors evaluating technical approaches

### Audience Characteristics
- Technically literate (can read code examples)
- Startup mindset (move fast, iterate)
- Budget-conscious but quality-focused
- Value developer experience and modern tooling
- Interested in Australian market specifics

## Voice & Tone

### Core Tone
- **Expert developer**: Speak developer-to-developer
- **Practical**: Code examples, real implementations
- **Modern**: Latest frameworks and best practices
- **Opinionated**: Take stances, share lessons learned
- **Startup-friendly**: Pragmatic, not perfectionist

### Writing Style
- Include code snippets and examples
- Reference specific versions and tools
- Share "lessons from the trenches"
- Balance ideal vs. pragmatic approaches
- Acknowledge technical debt tradeoffs

### Vocabulary
**Use**: implement, architecture, framework, performance, UX, iterate, ship, scale, modern, best practice
**Avoid**: simple, easy, basic, enterprise, legacy, corporate

## Content Structure

### Ideal Article Structure
1. **Problem/Challenge** (100-150 words): What we're solving
2. **Approach Overview** (150-200 words): Why this solution
3. **Implementation** (400-500 words): Detailed walkthrough with code
4. **Best Practices** (200-300 words): What we learned
5. **Performance/UX Considerations** (150-200 words): Making it great
6. **Conclusion** (100-150 words): Key takeaways

### Word Count Target
- **Minimum**: 1,200 words
- **Target**: 1,500 words
- **Maximum**: 1,800 words

- Target developer search queries

## Brand Don'ts

- Never write tutorial content without code examples
- Never recommend outdated frameworks or practices
- Never ignore performance and UX implications
- Never skip error handling in code examples
- Never be condescending to less experienced developers
- Never ignore the Australian market context when relevant
